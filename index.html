üèóÔ∏è Core Classes

SaveSlotsManager

Main component for managing multiple save slots and coordinating save/load operations.

CreateSaveSlot(string saveSlotName)

Creates a new save slot with the specified name.

Parameters:
  • saveSlotName - Unique identifier for the save slot

RegisterEveryObjectIntoSlot(string saveSlotName)

Automatically finds and registers all ISaveableData objects in the scene to the specified slot.

SaveToSaveSlot(string saveSlotName)

Saves all registered objects' data to the specified save slot with encryption and compression.

LoadFromSaveSlot(string saveSlotName)

Loads data from the specified save slot and applies it to registered objects.

bool IsValidSaveSlotName(string name)

Validates save slot name for file system compatibility and security.

Returns: True if name is valid, false otherwise.

DataPersistance

Handles individual save slot operations including encryption, compression, and file management.

Register(ISaveableData saveable)

Registers an object for saving/loading in this save slot.

bool SaveAll()

Saves all registered objects to encrypted, compressed file. Returns success status.

bool LoadAll()

Loads data respecting LoadOrder property. Returns success status.

bool ChangeName(string newName)

Safely renames the save slot and its associated files.

üîå Interfaces

ISaveableData

Interface that must be implemented by any class that wants to participate in the save system.

int LoadOrder { get; set; }

Determines the order in which objects are loaded. Lower numbers load first.

string GetSaveID()

Must return a unique identifier for this saveable object.

Example:
public string GetSaveID() => "PlayerStats_" + gameObject.GetInstanceID();

ü§ñ Auto Components

AutoSave

Automatically saves at specified intervals. Perfect for preventing data loss.

AutoLoad

Automatically loads all save slots on Start(). Useful for game initialization.

AutoRegister

Automatically registers all ISaveableData objects when scenes load.

üí° Complete Usage Examples

Basic Player Save System

public class PlayerController : MonoBehaviour, ISaveableData { public int LoadOrder { get; set; } = 0; [Saveable] public int level = 1; [Saveable] public float experience = 0f; [Saveable] public Vector3 position; private SaveSlotsManager saveManager; void Start() { saveManager = FindObjectOfType(); // Create save slot if it doesn't exist if (!saveManager.DoesSaveSlotExist("PlayerSave")) { saveManager.CreateSaveSlot("PlayerSave"); } // Register this object saveManager.RegisterObjectIntoSlot("PlayerSave", this); // Load existing data saveManager.LoadFromSaveSlot("PlayerSave"); } public string GetSaveID() => "Player_" + gameObject.name; // Save when player levels up public void LevelUp() { level++; experience = 0f; saveManager.SaveToSaveSlot("PlayerSave"); } }

Multi-Slot Game Save System

public class GameManager : MonoBehaviour { private SaveSlotsManager saveManager; void Start() { saveManager = FindObjectOfType(); // Create multiple save slots for (int i = 1; i <= 3; i++) { string slotName = "GameSlot" + i; if (!saveManager.DoesSaveSlotExist(slotName)) { saveManager.CreateSaveSlot(slotName); } } } public void SaveGame(int slotNumber) { string slotName = "GameSlot" + slotNumber; saveManager.RegisterEveryObjectIntoSlot(slotName); saveManager.SaveToSaveSlot(slotName); Debug.Log($"Game saved to slot {slotNumber}"); } public void LoadGame(int slotNumber) { string slotName = "GameSlot" + slotNumber; saveManager.LoadFromSaveSlot(slotName); Debug.Log($"Game loaded from slot {slotNumber}"); } }

üîí Security Features

  • AES Encryption: All save files are encrypted with unique keys
  • Key Management: Encryption keys are stored separately from save data
  • File Validation: Save slot names are validated against file system attacks
  • GZIP Compression: Reduces file size and adds obfuscation

‚ö° Performance Features

  • Load Order Management: Control initialization sequence
  • Selective Field Saving: Only marked fields are processed
  • Compression: Faster I/O with smaller files
  • Batch Operations: Efficient multi-object save/load

üéØ Best Practices

  • Use meaningful GetSaveID() implementations to avoid conflicts
  • Set appropriate LoadOrder for dependencies (managers load before dependents)
  • Mark only necessary fields with [Saveable] for performance
  • Use AutoSave component for critical data protection
  • Validate save slot names in UI to prevent errors
