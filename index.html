Overview

The Unity Save System is a comprehensive, modern save/load solution for Unity games. Built with JSON serialization at its core, it provides a clean API for saving game data with support for encryption, compression, and multiple save slots.

🚀 Easy to Use

Simple API with just Save() and Load<T>() methods. Get started in minutes with minimal setup required.

🔒 Secure

Built-in AES encryption support to protect save data from tampering and cheating.

📦 Compressed

GZIP compression reduces file sizes significantly, improving load times and storage efficiency.

🎮 Unity Ready

Full support for all Unity types including Vector3, Color, Quaternion, AnimationCurve, and more.

📁 Multi-Slot

Built-in save slot management allows multiple save files with easy switching between them.

⚡ Async

Modern async/await pattern prevents frame drops during save/load operations.

Quick Start

1. Add Setup Wizard

Add the SetupWizard component to any GameObject in your first scene and configure the settings in the inspector.

C# 
// Create or use existing save slot
if (!DataManager.DoesSlotExist("MainSave"))
{
    DataManager.CreateSaveSlot("MainSave");
}
DataManager.SetActiveSlot("MainSave");

2. Save Data

C# 
// Save simple values
DataManager.Save("PlayerLevel", 42);
DataManager.Save("PlayerName", "John Doe");
DataManager.Save("PlayerPosition", transform.position);

3. Load Data

C# 
// Load simple values
int level = DataManager.Load<int>("PlayerLevel");
string name = DataManager.Load<string>("PlayerName");
Vector3 position = DataManager.Load<Vector3>("PlayerPosition");

Features

Supported Data Types

  • Primitives: int, float, string, bool, etc.
  • Collections: List<T>, Dictionary<K,V>, arrays
  • Unity Types: Vector2/3/4, Quaternion, Color, Rect, Bounds, etc.
  • Custom Classes: Any serializable class or struct
  • Advanced Unity: AnimationCurve, Gradient, Matrix4x4

Save Slot Features

  • Multiple save slots with unique names
  • Runtime save slot creation/deletion
  • Save slot renaming
  • Automatic slot switching

Performance Features

  • Async operations to prevent frame drops
  • GZIP compression for smaller file sizes
  • Atomic write operations to prevent corruption
  • Efficient JSON serialization

DataManager API

The DataManager is the main entry point for the save system. It provides a simple, static API for all save/load operations.

Method Description Example
Save(string key, object value) Saves data to the active save slot DataManager.Save("Score", 1000);
Load<T>(string key) Loads data from the active save slot int score = DataManager.Load<int>("Score");
CreateSaveSlot(string name) Creates a new save slot DataManager.CreateSaveSlot("Player1");
DeleteSaveSlot(string name) Deletes an existing save slot DataManager.DeleteSaveSlot("Player1");
RenameSaveSlot(string oldName, string newName) Renames an existing save slot DataManager.RenameSaveSlot("Old", "New");
SetActiveSlot(string name) Changes the active save slot DataManager.SetActiveSlot("Player2");
DoesSlotExist(string name) Checks if a save slot exists bool exists = DataManager.DoesSlotExist("Player1");
GetSaveSlots() Returns all available save slots List<SaveSlot> slots = DataManager.GetSaveSlots();

SaveSlot Class

The SaveSlot class represents an individual save file. Each slot can have its own configuration set through the Setup Wizard.

Properties

Property Type Description
Name string The display name of the save slot
SavePath string Full file path to the save file

Methods

Method Description
Save(string key, object value) Saves data to this specific slot
Load<T>(string key) Loads data from this specific slot
Rename(string newName) Renames the save slot and updates file path
GetFileSize() Returns the size of the save file in bytes
GetLastModified() Returns when the save file was last modified

Setup Wizard

The SetupWizard component configures global settings for the save system. Add it to a GameObject in your first scene and configure the settings in the inspector.

Configuration Options

Setting Options Description
LogLevel None, Minimal, Detailed Controls how much logging information is displayed
Compression None, GZIP Compression method applied to all save operations
Encryption None, AES Encryption method applied to all save operations

Encryption

The EncryptionUtility class provides AES encryption functionality to protect save data from tampering. Encryption is configured globally through the Setup Wizard and applied automatically to all save operations when enabled.

Note: Encryption settings are configured through the Setup Wizard component and applied via dependency injection to all save operations.

Compression

The GZIP class provides compression functionality to reduce save file sizes. Compression is configured globally through the Setup Wizard and applied automatically to all save operations when enabled.

Note: Compression settings are configured through the Setup Wizard component and applied via dependency injection to all save operations.

Basic Examples

Save Player Progress

C# 
// Save player level
DataManager.Save("PlayerLevel", currentLevel);

// Load player level
int level = DataManager.Load<int>("PlayerLevel");

Save Custom Objects

C# 
// Save custom class
PlayerData playerData = new PlayerData { Health = 100, Experience = 1500 };
DataManager.Save("PlayerData", playerData);

// Load custom class
PlayerData loadedData = DataManager.Load<PlayerData>("PlayerData");

Save Unity Types

C# 
// Save Unity types
DataManager.Save("PlayerPosition", transform.position);
DataManager.Save("PlayerColor", playerRenderer.material.color);

// Load Unity types
Vector3 position = DataManager.Load<Vector3>("PlayerPosition");
Color color = DataManager.Load<Color>("PlayerColor");

Save Collections

C# 
// Save list
List<string> inventory = new List<string> { "Sword", "Shield", "Potion" };
DataManager.Save("Inventory", inventory);

// Load list
List<string> loadedInventory = DataManager.Load<List<string>>("Inventory");

Manage Save Slots

C# 
// Create and switch to new save slot
DataManager.CreateSaveSlot("NewGame");
DataManager.SetActiveSlot("NewGame");

// Check if slot exists
if (DataManager.DoesSlotExist("Player1"))
{
    DataManager.SetActiveSlot("Player1");
}

// Rename a save slot
DataManager.RenameSaveSlot("OldName", "NewName");

Complete API Reference

DataManager Methods

Method Returns Parameters Description
Save(string, object) void key, value Saves a value to the active save slot
Load<T>(string) T key Loads a value from the active save slot
CreateSaveSlot(string) void name Creates a new save slot with the given name
DeleteSaveSlot(string) void name Deletes the save slot and its file
RenameSaveSlot(string, string) void oldName, newName Renames an existing save slot
SetActiveSlot(string) void name Changes which slot is used for Save/Load operations
GetSaveSlots() List<SaveSlot> none Returns all available save slots
DoesSlotExist(string) bool name Checks if a save slot with the given name exists
